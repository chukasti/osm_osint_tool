# OSM OSINT Parser

Скрипт для сбора открытой информации об организациях в заданном районе или городе на основе данных [OpenStreetMap](https://www.openstreetmap.org/).

## Что собирает

- Веб-сайт организации
- Номер телефона
- E-mail
- Адрес (город, улица, номер дома, индекс)

## Установка

```bash
git clone https://github.com/chukasti/osm_osint_tool.git
cd osm_osint_tool
pip install requests
```

## Использование

```bash
./main.py ["название района"] [атрибут]
```

| Аргумент | Описание | Обязателен |
|---|---|---|
| `"название района"` | Район или город на любом языке | Да |
| `атрибут` | Приоритетный атрибут поиска: `website` / `email` / `phone` | Нет (по умолчанию `website`) |

### Примеры

```bash
# Найти все организации с сайтом в районе Хамовники
./main.py "Хамовники, Москва"

# Искать в первую очередь по email
./main.py "Хамовники, Москва" email

# По номеру телефона
./main.py "Невский район, Санкт-Петербург" phone
```

## Вывод

После выполнения скрипт печатает статистику и сохраняет результат в JSON-файл:

```
Всего объектов: 96
С сайтом:       96
С телефоном:    43
С email:        12
Данные записаны в файл Москва_Хамовники_21-05-2026_14-30-00
```

Пример содержимого файла:

```json
[
  {
    "name": "Кофейня у Льва",
    "website": "https://example.ru",
    "phone": "+7 495 000-00-00",
    "email": "info@example.ru",
    "addr:city": "Москва",
    "addr:street": "Льва Толстого",
    "addr:housenumber": "16"
  }
]
```

## Связка с другими инструментами

Собранные сайты организаций можно использовать совместно с **Shodan** для сопоставления домена с физическим местоположением и инфраструктурой:

```
site:example.ru → IP → Shodan → открытые порты, сервисы, геолокация
```

## Источники данных

- [OpenStreetMap](https://www.openstreetmap.org/) — геоданные объектов
- [Nominatim](https://nominatim.openstreetmap.org/) — геокодирование названия района в area_id
- [Overpass API](https://overpass-api.de/) — выборка объектов по тегам
